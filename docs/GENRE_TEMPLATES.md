# 类型小说玩法模板

本文档记录不同类型小说对应的 RPG 数值、场景交互和玩法循环。Skill 后续应根据原作类型自动选择模板。

## 通用规则

- 类型模板只提供玩法骨架，不替代原作风格。
- 可见数值最多 5 个。
- 隐藏数值可以用于条件、routes、结局，但不要在普通 UI 中暴露。
- 每个选择必须先有文本回响，再有数值反馈。
- 数值变化必须能被剧情解释。

## 修仙

### 核心体验

境界成长、灵力资源、功法选择、机缘探索、因果/道心、突破风险。

### 核心循环

```text
入门/当前困境
  -> 修炼/悟道
  -> 资源选择
  -> 外出历练
  -> 人情因果
  -> 突破/失败
  -> 新境界或新危机
```

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `realm` | 境界 | 可见 |
| `cultivation` | 修为进度 | 可见 |
| `qi` | 灵力 | 可见 |
| `daoHeart` | 道心 | 可见 |
| `comprehension` | 悟性 | 可见或隐藏 |
| `karma` | 因果/业力 | 隐藏 |
| `tribulationRisk` | 突破风险 | 隐藏 |
| `sectReputation` | 宗门声望 | 可见 |
| `masterTrust` | 师父信任 | 可见 |

### 推荐交互

- 打坐修炼。
- 内视丹田。
- 参悟功法。
- 探查灵气。
- 炼丹。
- 试剑。
- 接宗门任务。
- 突破境界。

### 模板要求

- 每章至少 1 个修炼或悟道互动。
- 每章至少 1 个资源/法宝/丹药选择。
- 每章至少 1 个道心或因果选择。
- 至少 1 个突破 routes 节点。

## 无限恐怖 / 副本生存

### 核心体验

任务压力、团队信任、资源消耗、恐惧/理智、情报收集、支线奖励、回归结算。

### 核心循环

```text
主神空间
  -> 任务发布
  -> 进入恐怖世界
  -> 搜索情报
  -> 团队分歧
  -> 危机爆发
  -> 支线奖励/逃生抉择
  -> 回归结算
  -> 强化兑换
```

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `hp` | 生命 | 可见 |
| `sanity` | 理智 | 可见 |
| `points` | 奖励点 | 可见 |
| `teamTrust` | 团队信任 | 可见 |
| `danger` | 区域危险度 | 可见 |
| `clue` | 情报完成度 | 可见 |
| `timeLeft` | 剩余时间 | 可见 |
| `infection` | 感染/诅咒 | 隐藏或可见 |

### 推荐交互

- 搜索房间。
- 观察怪物规律。
- 检查尸体。
- 分配队伍。
- 使用道具。
- 救或放弃队友。
- 冒险刷支线。
- 回主神空间兑换强化。

### 模板要求

- 每个任务世界必须有明确目标。
- 必须有时间或危险度压力。
- 至少 3 个可调查场景。
- 至少 1 个团队信任变化。
- 至少 1 个支线奖励诱惑。
- 必须有回归结算或任务失败节点。

## 悬疑推理

### 核心体验

线索收集、证词矛盾、嫌疑关系、推理完整度、错误推断成本。

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `deduction` | 推理完整度 | 可见 |
| `clueCount` | 线索数量 | 可见 |
| `pressure` | 调查压力 | 可见 |
| `suspectAlert` | 嫌疑人警觉 | 隐藏 |
| `mistake` | 错误推断次数 | 可见 |

### 推荐交互

- 调查现场。
- 询问人物。
- 追问。
- 比对证词。
- 提交推理。
- 指认证词矛盾。

### 模板要求

- 至少 3 条可收集线索。
- 至少 1 个需要 flag 或 clue 条件的推理选项。
- 错误推断应造成后果。

## 末世生存

### 核心体验

食物、水、弹药、感染、庇护所安全度、队伍士气、昼夜循环。

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `hp` | 生命 | 可见 |
| `food` | 食物 | 可见 |
| `water` | 水 | 可见 |
| `ammo` | 弹药 | 可见 |
| `shelter` | 庇护所安全 | 可见 |
| `morale` | 队伍士气 | 可见 |
| `infection` | 感染 | 可见或隐藏 |
| `noise` | 噪音 | 隐藏 |

### 推荐交互

- 搜刮资源。
- 加固门窗。
- 分配物资。
- 侦察周边。
- 夜晚防守。
- 处理感染者。

### 模板要求

- 每个阶段至少 1 次资源消耗。
- 至少 1 个物资分配导致关系变化。
- 至少 1 个资源不足导致的替代路线。

## 宫斗 / 权谋

### 核心体验

宠信、疑心、情报、把柄、派系关系、名声、利益交换。

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `favor` | 宠信 | 可见 |
| `suspicion` | 疑心 | 可见 |
| `reputation` | 名声 | 可见 |
| `silver` | 银钱 | 可见 |
| `intel` | 情报 | 可见 |
| `queenHostility` | 皇后敌意 | 可见 |
| `factionPower` | 派系力量 | 可见 |

### 推荐交互

- 送礼。
- 打探。
- 试探。
- 布置眼线。
- 交换利益。
- 保人或弃子。

### 模板要求

- 至少 3 个势力或关键人物关系。
- 每个关键选择至少影响一个关系数值。
- 高收益选择必须增加疑心或树敌风险。

## 普通文学 / 现实向

### 核心体验

保持原作气质，避免强行游戏化。只在必要时使用轻量状态。

### 推荐变量

- `trust`
- `distance`
- `memory`
- `resolve`
- `pressure`

### 模板要求

- 不强制使用 RPG 面板。
- 可只使用 `val` 和少量隐藏变量。
- 选择反馈以文本回响为主，数值反馈为辅。
