# 类型小说玩法模板

本文档记录不同类型小说对应的 RPG 数值、场景交互和玩法循环。Skill 后续应根据原作类型自动选择模板。

## 通用规则

- 类型模板只提供玩法骨架，不替代原作风格。
- 可见数值最多 5 个。
- 隐藏数值可以用于条件、routes、结局，但不要在普通 UI 中暴露。
- 每个选择必须先有文本回响，再有数值反馈。
- 数值变化必须能被剧情解释。
- **选择重量**：关键分歧选择（`weight=critical`）每章不超过 1 个，`weight=branch` 的选择每章不超过 3 个。critical 选择应当影响路线走向或结局条件；branch 选择提供次要分歧但不应拆分主线。
  - 可使用 `weightTag` 字段在选择旁边显示标签（如"关键抉择"、"支线影响"），增强视觉提示。
  - `weight=critical` 的选择应使用 `importantFlag` 标记，让玩家感知选择的重要性，播放器会短暂浮出提示"重要决定已记录"。
- **渐进探索**：每个节点最多配置 1 个 `surface` + 1 个 `deep` + 1 个 `ultimate` 交互。surface 为表层行为（一看即得），deep 为深入行为（需要观察或思考），ultimate 为终极发现（需要前置条件或多次交互积累）。
- **里程碑**：每个类型必须至少包含 1 个 `large` 里程碑（核心成长节点），用于标志玩家在剧情中的关键性进展。里程碑分为 small / medium / large 三级，均可配置 VFX 特效。
- **三层后果**：`weight=critical` 的选择必须配置至少 1 个 `delayedChanges`（延迟后果），即选择的真正影响不会立即显现，而是在后续章节中以出人意料的方式爆发。
- **结局设计**：每个类型至少 4 个结局（1 true + 1 dark + 1 neutral + 1 hidden），结局总数不超过 8 个。结局由数值条件、flag 和关键选择的综合结果判定。

## 修仙

### 核心体验

境界成长、灵力资源、功法选择、机缘探索、因果/道心、突破风险。

### 核心循环

```text
入门/当前困境
  -> 修炼/悟道
  -> 资源选择
  -> 外出历练
  -> 人情因果
  -> 突破/失败
  -> 新境界或新危机
```

### 节奏设计

```text
章节开头（探索与积累）
  → surface 交互（环境感知、基础信息获取）
  → minor 选择（日常决策，建立节奏感）
  → 小额数值变化（修炼、收集资源）

章节中段（紧张与冲突）
  → branch 选择（支线分歧，提供深度）
  → deep 交互（深入探索，获取关键信息或物品）
  → delayedChanges 触发（之前选择的即时后果显现）
  → 数值压力增加

章节高潮（关键抉择）
  → critical 选择（章节核心决策点）
  → 即时后果（第一层回响 + 第二层 delayedChanges）
  → 状态大幅变化或里程碑触发

章节收束（回味与悬念）
  → medium/small milestone 触发（如果本章达成成长节点）
  → 悬念文本（为下一章铺垫）
  → 状态面板回顾（让玩家感知章节间变化）
```

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `realm` | 境界 | 可见 |
| `cultivation` | 修为进度 | 可见 |
| `qi` | 灵力 | 可见 |
| `daoHeart` | 道心 | 可见 |
| `comprehension` | 悟性 | 可见或隐藏 |
| `karma` | 因果/业力 | 隐藏 |
| `tribulationRisk` | 突破风险 | 隐藏 |
| `sectReputation` | 宗门声望 | 可见 |
| `masterTrust` | 师父信任 | 可见 |

### 推荐交互

- 打坐修炼。
- 内视丹田。
- 参悟功法。
- 探查灵气。
- 炼丹。
- 试剑。
- 接宗门任务。
- 突破境界。

### 推荐里程碑

#### small

- 首次修炼：完成第一次打坐冥想，感应灵气入体。
- 首次获胜：在切磋或外门小比中首次战胜对手。
- 获得法宝：第一次获得法宝或灵器。
- 结交道友：与一位重要同辈建立友谊或同盟关系。

#### medium

- 突破小境界：在同一大境界内完成小阶段突破（如炼气三层到五层）。
- 领悟功法：深入理解一门功法的奥义，解锁新的技能或用法。
- 宗门大比晋级：在宗门考核或大比中成功晋级，获得更高权限或资源。

#### large

- 大境界突破：完成跨境界突破（如筑基成功、金丹凝结、元婴出窍），修为发生质变。
- 飞升：渡过天劫，离开当前世界位面。

#### VFX

- large 里程碑触发 `breakthrough` 特效：灵气粒子从角色周身上升飘散，屏幕边缘泛光，配合音效营造超脱之感。

### 推荐结局

| 结局 | 名称 | 触发条件 |
| --- | --- | --- |
| true | 飞升成仙 | 高道心 + 高修为 + 突破成功 |
| dark | 堕入魔道 | 低道心 + 杀戮过多 |
| neutral | 凡尘情缘 | 放弃修仙，选择红尘 |
| noble | 以身殉道 | 为保护宗门或道侣牺牲（高因果 + 关键选择） |
| hidden | 逆天改命 | 满足特殊条件 + 古道法 + 特殊机缘（隐藏数值达阈值） |
| romance | 仙侣双修 | 道心 + 感情双满 + 双方存活 |

### 渐进探索示例

以"修炼室"节点为例：

- **surface**：打坐修炼。玩家进行基础打坐操作，获得少量修为进度，感受灵气流动。无前置条件。
- **deep**：内视丹田。选择深入冥想，查看丹田内灵气分布状态，发现经脉中的细微阻塞或异常灵气波动。需至少完成 1 次 surface 交互。
- **ultimate**：发现功法秘密。通过深度内视，意外触碰到功法传承中的隐藏记忆碎片，揭示功法的真实来历或隐藏的修炼路径。需完成 deep 交互 + 悟性达到阈值。

### 模板要求

- 每章至少 1 个修炼或悟道互动。
- 每章至少 1 个资源/法宝/丹药选择。
- 每章至少 1 个道心或因果选择。
- 至少 1 个突破 routes 节点。
- 每章至少 1 个 `weight=critical` 选择（可跨章共享，即连续章节可共享同一个 critical 选择的后果阶段），并使用 `importantFlag` 标记。
- 每章至少 2 个 `surface` 交互 + 可选 `deep` / `ultimate`。
- 每个 Demo 至少包含 1 个 `large` 里程碑 + 2 个 `medium` + 3 个 `small`。
- 每个 Demo 结局总数 4-8 个。
- 关键选择应使用 `weightTag` 字段增强视觉提示。

### 安全屋章节

**节奏作用**：每 3-5 章设计一个安全屋章节，让玩家在紧张节奏后得到休息和整理。

**设计原则**：
- 无时间压力，无失败风险
- 不包含 weight="critical" 的选择
- 交互自由，无正确/错误之分
- 叙事节奏放缓，提供世界观补充或角色互动
- 数值变化仅限小额奖励（不影响关键数值）

**修仙示例**：
```
章节：山间小憩
- surface：与师兄闲聊（世界观补充）
- surface：整理丹房（发现小额奖励）
- branch：选择休息方式（不影响主线）
- 选择：继续修炼（唯一出口）
```

**无限恐怖示例**：
```
章节：主神空间休整
- surface：兑换强化（无风险）
- surface：与队友交流（团队信任微调）
- branch：选择下一个副本（分支影响）
```

**悬疑示例**：
```
章节：咖啡时间
- surface：整理线索笔记（无新线索）
- surface：与同事闲聊（角色关系）
- branch：选择调查方向（分支影响）
```

**末世示例**：
```
章节：营地日常
- surface：修理装备（小额资源）
- surface：与幸存者聊天（士气微调）
- branch：分配明日任务（分支影响）
```

**宫斗示例**：
```
章节：宫廷宴饮
- surface：品尝美食（无风险）
- surface：观察宾客（情报微调）
- branch：选择交谈对象（分支影响）
```

## 无限恐怖 / 副本生存

### 核心体验

任务压力、团队信任、资源消耗、恐惧/理智、情报收集、支线奖励、回归结算。

### 核心循环

```text
主神空间
  -> 任务发布
  -> 进入恐怖世界
  -> 搜索情报
  -> 团队分歧
  -> 危机爆发
  -> 支线奖励/逃生抉择
  -> 回归结算
  -> 强化兑换
```

### 节奏设计

```text
任务发布阶段（信息输入）
  → mission 目标展示（明确副本目标和时间限制）
  → surface 交互（初入恐怖世界的环境感知）
  → minor 选择（队伍分工、行动路线的初步决策）
  → 数值初始化（danger、timeLeft 设定）

探索阶段（情报收集与压力累积）
  → surface/deep 交互（搜索房间、检查线索、观察怪物规律）
  → branch 选择（支线诱惑：冒险刷分 vs 保守推进）
  → teamTrust 变化（队友间的意见分歧和信任博弈）
  → danger 逐步攀升（环境异变、怪物逼近的暗示）

危机爆发阶段（高压冲突）
  → critical 选择（生死抉择：救队友、放弃任务、冒险突围）
  → 即时后果（hp/sanity 大幅波动、队友状态变化）
  → delayedChanges 触发（之前探索中遗漏的情报后果显现）
  → timeLeft 倒计时压力

结算阶段（回归与反思）
  → medium/small milestone 触发（存活归来、支线完成）
  → points 结算（奖励点数和兑换选择）
  → 回顾面板（队友存活状态、本副本表现总结）
  → 悬念铺垫（下一个副本的暗示或预告）
```

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `hp` | 生命 | 可见 |
| `sanity` | 理智 | 可见 |
| `points` | 奖励点 | 可见 |
| `teamTrust` | 团队信任 | 可见 |
| `danger` | 区域危险度 | 可见 |
| `clue` | 情报完成度 | 可见 |
| `timeLeft` | 剩余时间 | 可见 |
| `infection` | 感染/诅咒 | 隐藏或可见 |

### 推荐交互

- 搜索房间。
- 观察怪物规律。
- 检查尸体。
- 分配队伍。
- 使用道具。
- 救或放弃队友。
- 冒险刷支线。
- 回主神空间兑换强化。

### 推荐里程碑

#### small

- 首次存活：在第一个恐怖场景中活过第一夜或第一波攻击。
- 获得关键道具：找到对当前副本或后续副本有重要影响的道具。
- 队友信任提升：通过关键行动获得至少一名核心队友的信任。

#### medium

- 完成支线任务：在不影响主线目标的前提下完成额外奖励任务。
- 单副本全队存活：整个副本过程中无人死亡。
- 发现Boss弱点：通过情报收集找到Boss的致命弱点或通关捷径。

#### large

- 存活归来：完成副本回归主神空间，进行结算并获得奖励。
- S级评价：以最高评价完成副本，获得超额奖励和稀有兑换资格。

#### VFX

- large 里程碑触发 `tearIn` 特效：画面从中心向外撕裂，现实世界透过裂缝显现，营造从恐怖世界"撕回"主神空间的视觉冲击。

### 推荐结局

| 结局 | 名称 | 触发条件 |
| --- | --- | --- |
| true | 唯一幸存者 | 全队阵亡但主角存活 + 完成主线任务 + 高理智 |
| dark | 深渊凝视 | 主角存活但理智归零，精神崩溃 |
| noble | 牺牲救赎 | 主角死亡但全队存活 + 高团队信任 |
| hidden | 全员存活 | 所有队友存活 + S级评价 + 完成所有支线（极稀有） |

### 渐进探索示例

以"恐怖房间"节点为例：

- **surface**：环顾四周。玩家查看房间基本布局，了解可用的家具、门窗、光源。无前置条件。
- **deep**：检查暗门。仔细检查墙壁、地板和家具背后的隐蔽结构，发现隐藏的通道或密室入口。需完成 1 次 surface 交互。
- **ultimate**：发现安全屋密码。在暗门后的密室中找到前人留下的密码本或线索，获得通往安全屋的关键信息。需完成 deep 交互 + 情报完成度达到阈值。

## 悬疑推理

### 核心体验

线索收集、证词矛盾、嫌疑关系、推理完整度、错误推断成本。

### 核心循环

```text
案件发生
  -> 现场调查
  -> 询问证人
  -> 比对证词
  -> 锁定嫌疑
  -> 推理验证
  -> 揭示真相/误判
```

### 节奏设计

```text
案件发生阶段（悬念铺垫）
  → 场景描写（案件现场的沉浸式描述）
  → surface 交互（拍照记录、基础物证收集）
  → minor 选择（初步调查方向选择）
  → pressure 初始设定（外部压力：时限、舆论、上级催促）

线索拼凑阶段（信息整合与排除）
  → surface/deep 交互（调查现场、比对证词、追问嫌疑人）
  → branch 选择（线索优先级判断：追查时间线 vs 人际关系 vs 物证链）
  → clueCount 增加 / mistake 累积（错误推断带来的成本）
  → suspectAlert 隐藏变化（玩家逼近真相时嫌疑人开始反常行为）

推理交锋阶段（高压对峙）
  → critical 选择（指认凶手 / 提交推理 / 保留证据）
  → 即时后果（推理正确则真相大白，推理失误则嫌疑人逃脱或嫁祸）
  → delayedChanges 触发（之前遗漏的线索或错误的追问在此刻显现影响）
  → deduction 大幅变化

真相收束阶段（回味与反转余地）
  → medium/small milestone 触发（锁定凶手范围、排除错误嫌疑）
  → 结局分支判定（根据 deduction、mistake 和关键 flag 确定走向）
  → 悬念文本（隐藏结局的暗示、未解之谜的余韵）
  → 状态面板回顾（线索完整度、推理正确率）
```

### 推荐变量

| 变量 | 含义 | 展示建议 |
| --- | --- | --- |
| `deduction` | 推理完整度 | 可见 |
| `clueCount` | 线索数量 | 可见 |
| `pressure` | 调查压力 | 可见 |
| `suspectAlert` | 嫌疑人警觉 | 隐藏 |
| `mistake` | 错误推断次数 | 可见 |

### 推荐交互

- 调查现场。
- 询问人物。
- 追问。
- 比对证词。
- 提交推理。
- 指认证词矛盾。

### 推荐里程碑

#### small

- 发现第一条线索：在现场或对话中获取第一份关键证据。
- 嫌疑人确认：初步锁定至少一名嫌疑人。

#### medium

- 锁定凶手范围：通过排除和交叉验证，将嫌疑人缩小到较小范围。
- 排除错误嫌疑：推翻一个看似合理的假设，避免错误指控。

#### large

- 揭开真相：拼凑完整证据链，揭示案件全貌和凶手动机。

#### VFX

- large 里程碑触发 `typewriterFlash` 特效：画面变为黑白打字机效果，真相文字逐字浮现，伴随机械敲击声，营造真相大白的高潮氛围。

### 推荐结局

| 结局 | 名称 | 触发条件 |
| --- | --- | --- |
| true | 完美破案 | 推理完整度满 + 错误推断为0 + 揭示动机 |
| dark | 局中人 | 主角自己就是凶手（隐藏数值+关键选择触发） |
| neutral | 悬案 | 线索不足，真相未能完全揭示，案件悬而未决 |

### 渐进探索示例

情报与布局阶段（暗中博弈）
  → deep 交互（偷听宫人闲谈、布置眼线、调查对手底细）
  → branch 选择（拉拢盟友 vs 按兵不动、交换利益 vs 保留把柄）

局势收束阶段（回味与布局）

## 普通文学 / 现实向

### 核心体验

保持原作气质，避免强行游戏化。只在必要时使用轻量状态。

### 推荐变量

- `trust`
- `distance`
- `memory`
- `resolve`
- `pressure`

### 模板要求

- 不强制使用 RPG 面板。
- 可只使用 `val` 和少量隐藏变量。
- 选择反馈以文本回响为主，数值反馈为辅。
- 每章至少 1 个 `weight=critical` 选择（可跨章共享），但 critical 选择的表现形式应自然融入叙事，不突兀。
