# 修仙 Demo AI 生成验证指南

**目的**：通过 AI 服务真实生成分支剧情 JSON，验证平台全流程可用性。

---

## 一、准备工作

### 1.1 环境要求

- Node.js 18+，`npm install` 已完成
- `.env.local` 已配置 AI API Key（硅基流动/DeepSeek/百炼/OpenAI 均可）
- `npm run dev` 可正常启动（localhost:3000）

### 1.2 素材准备

下方提供一段 800 字左右的修仙小说片段，可直接用于生成测试。你也可以替换为自己的素材。

## 二、修仙素材模板（可直接粘贴）

```
青炉峰下，少年林渊站在石阶前，仰头望着云雾缭绕的山门。这是他第三次参加青炉宗的入门考核。

"灵根测试，开始。"长老的声音从殿内传来。

林渊将手掌按在测灵石上。石面泛起微弱的青光——木灵根，下品。人群中传来窃笑声。下品灵根在修仙界几乎等同于凡人，终生难以突破炼气三层。

"下一位。"长老面无表情。

林渊却没有退下。他从怀中取出一块焦黑的铁片——那是他在山中采药时，从一道雷击的古树下拾得的。

"弟子斗胆，请求以'杂学'入宗。"林渊朗声道。

殿内一静。杂学弟子不修法力，专研丹药、符箓、阵法，地位远低于内门，但青炉宗百年前确有一位以杂学证道的祖师。

长老眯起眼："你可知杂学之路，千年无人走通？"

"弟子知晓。"林渊握紧了那块焦黑铁片，"但弟子也知，三日前的雷劫，并非天象。"

长老瞳孔骤缩。三日前那场覆盖整座青炉峰的雷劫，确实古怪——明明是晴空万里，却有九道紫雷劈在后山禁地。宗门高层秘而不宣，这少年是如何……

"你看到了什么？"长老的声音低了下来。

林渊低头看着手中铁片。那上面隐约有一道剑痕，像是有人以无上剑意，将什么东西封印在了铁片之中。雷劫之夜，他亲眼看见最后一道紫雷劈中古树后，有什么东西从树心飞出，钻入了这块铁片。

"弟子看到……"林渊深吸一口气，"雷中有人。"

殿内死寂。

最终，林渊以杂学弟子的身份入了青炉宗。他被分配到丹药阁，负责照看一炉最基础的"清心丹"。同门嘲笑他，说他一辈子就只能和这些低级丹药打交道。

但林渊每晚都会拿出那块焦黑铁片，对着月光端详。铁片上的剑痕在月光下会发出极淡的金色纹路，像是某种他看不懂的文字。

一个月后的深夜，林渊在丹药阁值夜。他照例取出铁片，却突然发现——丹炉中炼制失败的废丹残渣，滴落在铁片上时，那些金色纹路亮了起来。

"这是……"林渊瞪大眼睛。

金色纹路组成了一个字："启"。

与此同时，青炉宗后山禁地，传来了沉闷的轰鸣声。守夜的弟子看见禁地结界泛起涟漪，像是有什么东西，正在从内部苏醒。

林渊不知道的是，百年前那位以杂学证道的祖师，留下的最后一句话是："后世若有木灵根弟子，以废丹启封，可开后山剑冢。"

而那块焦黑铁片，正是剑冢的钥匙。
```

## 三、生成流程

### 步骤 1：打开即时体验页

访问 http://localhost:3000/instant

### 步骤 2：填写信息

| 字段 | 填写内容 |
|------|----------|
| 作品标题 | `青炉夜火`（或留空让 AI 自动生成） |
| 选择类型 | **修仙** |
| 启用 RPG 数值系统 | ✅ 勾选 |

### 步骤 3：配置规则（可选）

点击「自定义游戏规则」展开面板，建议配置：

| 规则 | 推荐值 | 说明 |
|------|--------|------|
| 节奏密度 | 舒缓 | 修仙题材适合充分叙述 |
| 数值影响强度 | 中度 | 影响选项可见性 |
| 隐藏内容比例 | 多 | 适合探索剑冢秘密 |
| 结局倾向 | 均衡 | HE/BE 都有 |
| 叙事人称 | 第二人称"你" | 沉浸感更强 |
| 对白密度 | 中 | 叙述与对话平衡 |
| NPC 关系网络 | ✅ 开启 | 师父/同门/神秘存在 |
| 时间压力 | ❌ 关闭 | 修仙不急 |

### 步骤 4：粘贴素材

将上方「修仙素材模板」全文粘贴到文本输入框。

### 步骤 5：点击「开始生成」

观察 SSE 流式进度：
1. `连接 AI...` → `正在生成剧本...` → `正在写入...`
2. 生成完成后自动跳转到 `/play/{id}`

### 步骤 6：验证生成结果

#### 6.1 JSON 结构检查

```bash
# 获取生成的作品 ID（从 URL 中）
curl http://localhost:3000/api/works

# 获取单个作品 JSON
curl http://localhost:3000/api/works/{id} > demo.json

# 校验
python story-to-game-source/scripts/validate.py demo.json
```

**期望结果**：无致命错误（可能有警告）。

#### 6.2 关键字段检查

打开 `demo.json` 检查以下字段是否存在：

```json
{
  "meta": {
    "title": "...",
    "genre": "xianxia",
    "rpg": { "enabled": true, "primaryStats": [...] },
    "rules": { "pacing": "relaxed", ... }
  },
  "nodes": { ... },           // 40-80 个节点
  "achievements": { ... },    // 5-10 个成就
  "milestones": [...],        // 3+ 个里程碑
  "endings": [...],           // 3-5 个结局
  "npcRelations": [...],      // 3+ 个 NPC（如果开启了 NPC 关系）
  "variables": { ... }        // RPG 状态变量
}
```

#### 6.3 播放器功能验证清单

在 `/play/{id}` 页面逐一验证：

- [ ] 状态栏显示正确的修仙数值（境界、修为、灵力等）
- [ ] 叙事文本正常显示，分段渲染
- [ ] 选项可点击，选择后有变化反馈
- [ ] 条件选项（需要特定数值）正确置灰/隐藏
- [ ] 权重标签（critical/branch/minor）视觉区分正确
- [ ] 场景切换动画正常
- [ ] 主题色为修仙风格（青绿色调）
- [ ] BGM 播放（Web Audio API 合成）
- [ ] SFX 播放（选择/数值变化/章节切换等）
- [ ] 背包系统（如有物品变更）
- [ ] 存档/读档功能
- [ ] 成就解锁提示
- [ ] NPC 关系面板（如开启了 NPC 关系）
- [ ] 多路线游玩（至少 3 条不同路线）

## 四、常见问题排查

| 问题 | 排查方法 |
|------|----------|
| AI 生成卡住/超时 | 检查 `.env.local` API Key 是否正确；检查网络连接；尝试缩短素材 |
| 生成 JSON 解析失败 | 查看终端 API 日志，可能是 AI 输出非纯 JSON |
| validate.py 报错 | 根据错误码修复（RPG-001~031），大部分是引用完整性问题 |
| 播放器白屏 | 检查浏览器控制台是否有 JS 错误；检查 iframe 加载是否正常 |
| 无声音 | 检查浏览器是否阻止了自动播放；点击页面任意位置后再测试 |
