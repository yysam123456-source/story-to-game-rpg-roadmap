# Story-to-Game 产品形态改革方案

**日期**：2026-07-06
**版本**：v4.1（轻量架构版）
**状态**：已确认方向

---

## 一、核心诊断

### 1.1 当前链路的问题

现有产品形态是"开源工具套件"，用户需要完成 8+ 步操作才能体验到最终成果：

```
发现项目 → 去 GitHub → 下载 SKILL.md → 打开 ChatGPT/Claude
→ 上传文件 → 写小说原文 → 等待 AI 生成 → 下载 JSON
→ 打开网页播放器 → 导入 JSON → 开始游玩
```

每一步都是用户流失点。没有普通用户会走完这条路。

### 1.2 产品形态的本质转变

从 **"给用户一套工具，让他自己组装"** 变成 **"用户只管创作和游玩，平台负责一切中间环节"**。

---

## 二、新形态：静态前端 + 轻量 API + JSON 文件存储

### 2.1 架构极简原则

不搞复杂的用户系统、不搞数据库、不搞微服务。三层解决一切：

```
┌─────────────────────────────────────────────┐
│              静态前端（HTML/CSS/JS）            │
│  沿用 rpg-game-ui-v2/ 现有架构               │
│  新增：首页 / 即时体验 / 创作者 / 作品库       │
└─────────────────────┬───────────────────────┘
                      │ HTTP API
┌─────────────────────▼───────────────────────┐
│          轻量后端（Python Flask）               │
│  POST /api/generate    → AI 生成 JSON         │
│  GET  /api/works       → 列出作品              │
│  GET  /api/works/:id   → 返回作品 JSON         │
│  GET  /api/status/:id  → 生成进度（SSE）       │
└─────────────────────┬───────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        │             │             │
┌───────▼──┐  ┌──────▼────┐  ┌────▼──────┐
│  AI API  │  │  files/   │  │ validate  │
│ Claude   │  │  *.json   │  │   .py     │
│ GPT-4o   │  │  文件存储  │  │  校验脚本  │
└──────────┘  └───────────┘  └───────────┘
```

### 2.2 用户旅程重构

**创作者链路（从 8 步缩短到 3 步）**：

```
打开网页 → 粘贴/上传小说 → AI 生成 → 预览/下载
```

**玩家链路（从 8 步缩短到 2 步）**：

```
打开网页 → 浏览作品 → 点击即玩
```

### 2.3 核心页面

| 页面 | 功能 | 前端基础 |
|------|------|----------|
| 首页 | 品牌展示 + "立即体验"入口 + 热门作品 | 新建 |
| 即时体验 | 左屏粘贴小说 → 右屏即时生成 + 游玩预览 | 新建 + game-main.html |
| 创作者 | 上传完整小说 → 选择类型 → AI 生成 → 预览/下载 | 新建 |
| 作品库 | 列表 + 分类筛选 + 点击即玩 | 新建 + game-main.html |
| 播放器 | 现有 rpg-game-ui-v2/ 完全复用 | game-main.html |

### 2.4 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 前端 | HTML/CSS/JS（沿用现有） | 不引入框架，保持轻量 |
| 后端 | Python Flask | 最轻量，与 validate.py 同语言 |
| AI | Claude 3.5 Sonnet（主）/ GPT-4o（备） | 长文本处理强，function calling 确保 JSON 格式 |
| 存储 | files/ 目录 + JSON 文件 | 零数据库，符合项目轻量原则 |
| 部署 | Vercel（前端）+ Railway/Fly.io（后端） | 低成本，自动部署 |

## 三、竞品差异化

| 差异点 | 我们 | 橙光/易次元 | AI Dungeon |
|--------|------|-------------|------------|
| 创作门槛 | 粘贴文本即可 | 学专用工具 | 无创作 |
| 叙事质量 | 结构化+专业分支设计 | 创作者自由 | AI 幻觉严重 |
| RPG 系统 | 类型模板自动配置 | 手动配置 | 无 |
| 分发 | 网页+小程序 | APP内 | 网页 |

## 四、商业化（Q4 启动）

- 创作者 Pro 订阅：解锁无限生成 + 高级主题 + 去水印
- 玩家内购：单部作品付费解锁完整剧情
- 平台抽成 20-30%
