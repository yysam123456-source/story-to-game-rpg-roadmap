# Story-to-Game 产品形态改革方案

**日期**：2026-07-06
**版本**：v4.2（Next.js 全栈版）
**状态**：已确认方向

---

## 一、核心诊断

### 1.1 当前链路的问题

现有产品形态是"开源工具套件"，用户需要完成 8+ 步操作才能体验到最终成果：

```
发现项目 → 去 GitHub → 下载 SKILL.md → 打开 ChatGPT/Claude
→ 上传文件 → 写小说原文 → 等待 AI 生成 → 下载 JSON
→ 打开网页播放器 → 导入 JSON → 开始游玩
```

每一步都是用户流失点。没有普通用户会走完这条路。

### 1.2 产品形态的本质转变

从 **"给用户一套工具，让他自己组装"** 变成 **"用户只管创作和游玩，平台负责一切中间环节"**。

---

## 二、新形态：Next.js 全栈平台

### 2.1 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Next.js 14 App Router                     │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
│  │   /      │ │ /instant │ │ /create  │ │ /library │       │
│  │  首页    │ │ 即时体验 │ │ 创作者   │ │ 作品库   │       │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           /play/[id] 播放器页                        │   │
│  │  (复用 rpg-game-ui-v2 组件迁移为 React)              │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  API Routes                                         │   │
│  │  /api/generate  → AI 生成 JSON (SSE 流式进度)       │   │
│  │  /api/works     → 作品列表                          │   │
│  │  /api/works/:id → 返回作品 JSON                     │   │
│  └─────────────────────────────────────────────────────┘   │
└──────────────────────────┬──────────────────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        │                  │                  │
┌───────▼──┐  ┌──────▼────┐  ┌────▼──────┐
│  AI API  │  │  Vercel   │  │  Local   │
│ Claude   │  │  Blob     │  │  files/  │
│ GPT-4o   │  │  Storage  │  │  (dev)   │
└──────────┘  └───────────┘  └───────────┘
```

### 2.2 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 框架 | Next.js 14 App Router | 用户现有技术栈，全栈能力，API Routes 内置 |
| 语言 | TypeScript | 类型安全，可维护性 |
| 样式 | Tailwind CSS + CSS 变量 | 快速开发，Design Tokens 易于维护 |
| UI 组件 | Shadcn/ui | 基于 Tailwind，可定制，无障碍支持 |
| 部署 | Vercel | 用户习惯，自动 CI/CD，Edge Network |
| 国内访问 | 备案域名 → Vercel | Vercel 支持自定义域名，国内有 CDN 节点 |
| 存储 | Vercel Blob Storage | 无服务器，与 Vercel 集成，JSON 文件直接存 |
| AI 调用 | API Routes 代理 | 前端不暴露 API Key，安全可控 |

### 2.3 用户旅程重构

**创作者链路（3 步）**：

```
打开 stg.io → 粘贴/上传小说 → AI 生成 → 预览/下载
```

**玩家链路（2 步）**：

```
打开 stg.io → 浏览作品 → 点击即玩
```

### 2.4 核心页面

| 路由 | 页面 | 功能 |
|------|------|------|
| `/` | 首页 | 品牌展示 + "立即体验"CTA + 热门作品 |
| `/instant` | 即时体验 | 左屏粘贴小说 → 右屏即时生成 + 游玩预览 |
| `/create` | 创作者 | 上传完整小说 → 选择类型 → AI 生成 → 预览/下载 |
| `/library` | 作品库 | 网格列表 + 分类筛选 + 点击即玩 |
| `/play/[id]` | 播放器 | 加载 JSON 游玩（复用 rpg-game-ui-v2 组件） |

## 三、可扩展性设计

### 3.1 模块化架构

- **播放器组件**独立封装，不依赖页面逻辑
- **AI 服务层**抽象接口，方便切换模型（Claude ↔ GPT ↔ 国内模型）
- **存储层**抽象接口，开发/生产环境自动切换
- **主题系统**CSS 变量驱动，新增题材只需加一套变量

### 3.2 未来扩展预留

| 功能 | 预留方案 |
|------|----------|
| 用户系统 | 不侵入现有代码，后续可加 NextAuth |
| 数据库 | 目前用 Blob Storage，后续可切 PostgreSQL |
| 小程序 | 复用同一套 JSON 协议，独立小程序项目 |
| 商业化 | Pro 订阅通过 Stripe，内购通过支付接口 |
| i18n | 文案全部用 i18n 函数包裹，预留多语言 |

## 五、部署方案

### 5.1 Vercel 部署（主）

```bash
# 开发
npm run dev

# 部署
vercel --prod
```

- 自动 CI/CD：git push → 自动构建 → 自动部署
- Edge Network：全球 CDN 加速
- Serverless Functions：API Routes 自动伸缩

### 5.2 国内访问优化

- 域名备案后，在 Vercel 设置自定义域名
- Vercel 在国内有 CDN 节点，访问速度可接受
- 如需更快：阿里云/腾讯云 CDN 回源到 Vercel
