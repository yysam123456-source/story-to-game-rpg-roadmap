# Story-to-Game 平台化路线图

**日期**：2026-07-06
**版本**：v4.0（平台化升级）
**定位**：在线 AI 互动叙事平台（从"开源工具套件"升级）
**核心原则**：静态前端 + 轻量 API + JSON 文件存储，无数据库、无用户系统

---

## 1. 产品定位

> **「在线 AI 互动叙事平台」—— 粘贴小说，AI 自动生成分支剧情游戏，即时游玩**

### 1.1 核心转变

| 维度 | 旧定位（工具套件） | 新定位（在线平台） |
|------|---------------------|---------------------|
| 产品形态 | 开源 HTML/JS 工具套件 | 在线平台 + API 服务 |
| 使用方式 | 下载 HTML → 手写 JSON → 本地运行 | 粘贴小说 → AI 生成 → 即时游玩 |
| 存储方案 | 无后端，本地文件 | 服务端 files/ 目录存 JSON |
| AI 能力 | SKILL.md 作为文档参考 | API 服务自动执行九步工作流 |
| 创作门槛 | 需理解 JSON 格式，手动编辑 | 上传小说即可，AI 全自动生成 |
| 部署方式 | GitHub Release 分发 | 自建站点 + 微信小程序 |

### 1.2 架构概览

```
┌─────────────────────────────────────────────────┐
│                    前端层                         │
│  rpg-game-ui-v2/ (HTML/CSS/JS)                   │
│  ├─ 首页（产品介绍 + CTA）                        │
│  ├─ 即时体验页（粘贴 → 生成 → 游玩）              │
│  ├─ 创作者页（上传小说 → 生成 → 预览/下载）        │
│  └─ 作品库页（列表 + 分类 + 点击即玩）              │
├─────────────────────────────────────────────────┤
│                    API 层（Flask）                 │
│  ├─ POST /api/generate    — AI 生成服务           │
│  ├─ GET  /api/works       — 作品列表              │
│  └─ GET  /api/works/:id   — 获取单个作品 JSON      │
├─────────────────────────────────────────────────┤
│                    AI 层                          │
│  Claude / GPT-4o API                              │
│  └─ 执行 SKILL.md 九步工作流                       │
├─────────────────────────────────────────────────┤
│                    存储层                          │
│  files/ 目录（JSON 文件）                          │
└─────────────────────────────────────────────────┘
```

### 1.3 不变的核心原则

以下原则从项目诞生起持续有效：

1. **文学优先**：每一个选择必须从当前场景自然长出，每一个后果必须被世界认真承接，每一个结局必须以判词收束
2. **轻 RPG 服务叙事**：数值变化先有文本回响，再展示数值，不压过文学体验
3. **JSON 可校验**：所有 JSON 必须通过 validate.py 的 18 条 RPG 校验规则
4. **向后兼容**：新增字段均为可选，旧 JSON 不受影响
5. **类型深度**：修仙/无限流/悬疑/末世/宫斗各有专属模板、状态系统、主题色和 VFX
6. **选择有重量**：choice.weight 四级系统 + delayedChanges 三层后果
7. **成长有仪式**：milestones 三级庆祝 + endings 多类型结局

## 3. Q3 2026：MVP 核心验证（7-9 月）

### 3.1 核心目标（一句话）

验证"AI 自动生成分支剧情 JSON + 在线即时游玩"的核心假设是否成立。

### 3.2 关键任务

| # | 任务 | 优先级 | 说明 |
|---|------|--------|------|
| 1 | Flask 后端 API 服务搭建 | P0 | 3 个核心端点：`/api/generate`、`/api/works`、`/api/works/:id` |
| 2 | AI 生成服务（快速模式） | P0 | 调用 Claude/GPT-4o API，执行 SKILL.md 快速模式流程，输出符合 SCHEMA_v1 的 JSON |
| 3 | 前端即时体验页 | P0 | 粘贴小说片段 → 一键生成 → 即时在浏览器中游玩 |
| 4 | 前端创作者页 | P0 | 上传小说 → AI 生成完整 JSON → 在线预览 / 下载 JSON 文件 |
| 5 | 前端作品库页 | P0 | 作品列表 + 按类型分类 + 点击即玩（加载 JSON 并启动 rpg-game-ui-v2 播放器） |
| 6 | SKILL.md RPG 扩展升级 | P1 | 对齐 SCHEMA_v1 规范，确保 AI 生成的 JSON 包含 meta.rpg、milestones、endings 等字段 |

### 3.3 交付标准（可验证）

- [ ] Flask 服务部署可访问，3 个 API 端点响应正常
- [ ] 即时体验页：粘贴 500 字以内小说片段，30 秒内生成可玩的分支剧情（快速模式）
- [ ] 创作者页：上传完整小说，生成 40-80 节点、3-5 结局的完整 JSON
- [ ] 作品库页：展示所有已生成作品，按类型筛选，点击即玩
- [ ] 生成的 JSON 通过 validate.py 全部 18 条 RPG 校验规则
- [ ] 修仙 Demo 通过 AI 服务真实生成（非手工编写），可完整游玩
- [ ] 前端播放器完整支持：状态栏、选择重量、变化反馈、条件选项置灰
- [ ] 前端新增首页（产品介绍、使用流程说明、CTA 按钮）

### 3.4 风险检查点

| 风险 | 概率 | 影响 | 缓解策略 | 触发条件 |
|------|------|------|----------|----------|
| AI 生成质量不稳定 | 高 | 高 | 快速模式先上，人工审核兜底；校验器拦截不合规 JSON | validate 通过率 < 70% |
| API 响应过慢 | 中 | 高 | 长文本拆分处理；设置 60 秒超时 + 进度提示 | 平均生成时间 > 60 秒 |
| AI API 成本过高 | 中 | 中 | 快速模式控制 token；设置每日生成上限 | 月成本 > 预算 2 倍 |
| 修仙 Demo 生成效果差 | 中 | 高 | 准备手工精编的 fallback Demo；迭代 prompt | 用户测试满意度 < 6/10 |
| 前后端联调问题 | 低 | 中 | API mock 先行；JSON 格式契约先行定义 | 联调阻塞 > 3 天 |

### 3.5 与上一个季度的依赖关系

- 依赖已完成：SCHEMA_v1 已锁定、validate.py 18 条规则就绪、rpg-game-ui-v2 播放器可用
- Q3 是全新方向的起点，从"纯前端工具"转向"在线平台"，无前置季度依赖

## 8. 利益相关者视图

### 8.1 产品视图

- **定位**：在线 AI 互动叙事平台
- **核心价值**：零门槛创作（粘贴小说即可）+ 即时游玩 + AI 全自动生成
- **差异化**：类型深度（5 种专属模板）+ 文学保真（九步工作流）+ 轻 RPG（服务叙事不压文学）
- **时间表**：Q3 MVP → Q4 分发 → Q1 生态 → Q2 商业化

### 8.2 技术视图

- **技术栈**：HTML/CSS/JS（rpg-game-ui-v2）+ Python Flask + Claude/GPT-4o API + JSON 文件存储
- **架构约束**：无数据库、无用户系统、静态前端 + 轻量 API
- **关键决策**：快速模式先行验证，再迭代完整模式；JSON 格式兼容 SCHEMA_v1
- **扩展路径**：JSON 文件存储 → 按需引入数据库；无用户系统 → 按需引入微信授权登录

### 8.3 设计视图

- **设计语言**：Glassmorphism 暗色系，5 种题材主题实时切换
- **设计原则**：文学优先，数值不压过内容
