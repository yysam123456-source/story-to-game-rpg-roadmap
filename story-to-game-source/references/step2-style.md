# 第二步：风格指纹提取

## 目标

从原作中提取具体的、可执行的风格规则。不是"模仿原作风格"这种模糊指令，而是一份写作时可以逐条检查的清单。

## 提取方法

### 1. 叙述温度

从原作中找 3-5 个代表性段落，判断叙述的情感距离。叙述温度不是文学流派标签，是一个实操判断：**这个作者在写到角色痛苦/快乐/愤怒时，镜头拉得多远？**

| 温度类型 | 特征 | 通俗理解 | 参考范例 |
|----------|------|----------|----------|
| 冷叙述 | 不带情感评价，像监控摄像头一样记录发生了什么 | 角色哭了，但叙述只写"她的眼睛湿了"，不写"她悲伤得无法自持" | 东野圭吾的犯罪叙述段、网文中"无情流"写法、海明威 |
| 克制抒情 | 有情感但被压住，不直说，靠一个动作或一个物件暗示 | 不写"他很想她"，写"他把她用过的杯子从柜子里拿出来又放回去了" | 《请回答1988》的叙事感、余华《活着》中的平静段落、日系轻小说中的留白 |
| 日常温感 | 叙述像聊天一样自然，不刻意冷也不刻意热，保持生活化的体温 | 该写感受就写，但不夸张、不煽情、不拔高，像在跟朋友复述一件事 | 大部分都市言情小说、青春校园文、现代生活剧 |
| 反讽 | 叙述表面平静甚至轻松，但内容本身荒诞或残酷，温差制造黑色幽默 | 用写菜谱的语气写一场谋杀 | 刘慈欣写社会场景时的冷幽默、余华《第七天》、网文中的"丧系吐槽流" |
| 热叙述 | 直接表达情感，修辞密度高，叙述者明显在替角色激动 | "他的心像被撕裂了一样"这种写法出现频率高 | 大部分仙侠/玄幻小说的高潮段、言情小说的情感爆发段、爽文 |
| 燃叙述 | 热叙述的变体，情绪指向亢奋/热血/燃烧，而非悲伤 | 节奏加快、排比密集、角色台词自带BGM感 | 少年漫改小说、竞技文高潮、战争/体育题材的决战段 |
| 暗涌型 | 表面是日常温感，但字里行间有不安、有压迫、有什么要发生 | 写的全是正常生活，但读者觉得不对劲 | 悬疑小说的铺垫段、心理惊悚类网文、紫金陈的日常段落 |

**注意：一部作品不会只有一种叙述温度。** 大部分作品是"基础温度+变化"的模式。比如一部悬疑小说的基础温度是"冷叙述"，但在角色情感爆发时切换到"克制抒情"。需要记录的是：基础温度是什么，在什么场景下温度会变化，变化幅度多大。

**输出格式**：

```
基础叙述温度：暗涌型
温度变化规则：
- 日常场景：日常温感，保持生活化
- 调查/推理场景：冷叙述，纯记录
- 角色对峙场景：切换到克制抒情，情绪通过动作暗示
- 真相揭露场景：短暂升温到反讽（用平静语气写残酷内容）
游戏正文指引：segments 默认使用短句、不带修饰的叙述；对白保持口语化；只在关键情绪节点允许出现一两句带感官细节的描写（如气味、温度），绝不出现直接情绪形容词。
```

### 2. 对白风格

分析原作中的对白，回答以下问题：

- 对白是口语化还是书面化？
- 平均句长多少？（短句急促 / 中等 / 长句绵延）
- 角色之间的对白是否有明显的风格差异？
- 对白中是否有潜台词？（说的和想的不一样）
- 是否使用方言、行话、特殊用语？
- 对白的密度如何？（大段叙述中偶尔对白 / 以对白为主 / 交替进行）

**输出**：一份对白规则表，每个主要角色的对白规则单独列出。格式：

```
【角色名】的对白规则：
- 句长：通常不超过X个字
- 口语程度：[高/中/低]
- 特殊用词/口头禅：XXX
- 潜台词倾向：[经常/偶尔/几乎没有]
- 禁止出现：[过度文学化的表达 / 感叹号泛滥 / 等]
```

### 3. 节奏型

分析原作的句子节奏。取 3-5 个代表性段落，统计：

- 句子的平均长度（字数）
- 长短句的交替模式
- 段落之间的呼吸感（是紧凑连续还是有大量留白）
- 场景转换的速度（快切 / 渐变 / 跳跃）

**输出**：节奏规则，描述在 segments 中应该如何控制句长和段落节奏。

### 4. 视角与人称转换规则

确定原作的叙事视角，制定转换策略：

**原作第一人称 → 游戏人称：**
- 通常保持"我"
- 叙述段用"我"的内视角
- 限制：玩家只能知道"我"知道的信息
- 对白中其他角色的信息通过对白传递

**原作第三人称有限视角 → 游戏人称：**
- 转为"你"
- 叙述段从外部描述"你"的动作和感知
- 限制：不写"你想到了"、"你意识到"这类心理描写
- 用"你看到"、"你听到"、"你注意到"替代

**原作第三人称全知视角 → 游戏人称：**
- 转为"你"，但信息披露策略要大改
- 原作中读者通过全知视角知道的其他角色的想法/行动，在游戏中不能直接告诉玩家
- 这些信息需要通过：对白泄露、物件/线索发现、NPC 行为暗示等方式间接传递
- 这反而创造了天然的分支空间：玩家是否发现了某条线索

**原作多视角 → 游戏人称：**
- 选定一个主要角色作为玩家视角
- 其他视角角色的信息全部改为间接获取
- 可以在特定分支中切换视角（用 chapterTitle 标记视角切换）

### 5. 原作禁区

列出原作风格中"从来不做"的事情。这些禁区在游戏正文中同样禁止：

常见禁区类型：
- 不直接描写角色心理活动
- 不使用感叹号
- 不使用过度修辞（比喻、拟人等）
- 不进行道德评判
- 不煽情
- 不解释主题
- 不使用特定类型的形容词

**输出**：一份禁止清单，格式为"在本作的游戏改编中，绝对不能出现：……"

## 风格锚点文档的最终格式

```markdown
# 风格锚点：《作品名》

## 叙述温度
（一句话定义 + 执行指引）

## 对白规则
### 通用规则
（适用于所有角色的对白基础规则）
### 角色对白规则
（每个主要角色单独列出）

## 节奏规则
（句长、段落节奏、场景转换速度的具体要求）

## 视角转换
（原作人称 → 游戏人称的完整转换方案）

## 禁止清单
（绝对不能出现的内容/表达方式）
```

## 风格自检

写完风格锚点后，自检：

- [ ] 每一条规则是否具体到可以用来检查一段正文？（"保持原作风格"这种不算）
- [ ] 对白规则是否能区分不同角色？
- [ ] 禁止清单是否遗漏了原作的重要特征？
- [ ] 视角转换方案是否考虑了信息披露的变化？
