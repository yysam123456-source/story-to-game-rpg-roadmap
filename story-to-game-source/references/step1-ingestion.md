# 第一步：原作摄入与记忆索引构建

## 目标

将原作全文压缩为结构化索引，使后续步骤无需携带原文全文即可精确工作。

## 索引的三层结构

### 骨架层：事件链（细粒度拆解）

**不要只拆"大事件"。** 把原作拆到动作级别的小事件——每一个独立的动作、对话轮次、发现、移动、情绪变化都是一个事件。小说中一个段落可能包含 3-5 个小事件。

拆解粒度参考：
- ❌ 粗粒度（不够）："主角去了天台" — 这里至少包含"做出决定→起身→走出房间→上楼梯→到达天台"五个动作
- ✅ 细粒度："主角放下信→站起来→走到门口→打开门→沿楼梯上行→推开天台的门→站在天台上"

每个事件包含：

```
事件编号 | 时间标记 | 地点 | 涉及角色 | 事件摘要 | 因果关系 | 情绪基调 | 叙事目的
```

**叙事目的**是这个事件在原作中为什么必须存在——如果删掉它，故事会缺少什么。每个事件至少一个叙事目的，最多三个。

叙事目的的类型：
- **交代**：交代人物/空间/时间/规则/世界观信息
- **建立**：建立或强化角色特征、角色关系
- **推进**：推进情节、制造冲突、升级冲突
- **伏笔**：埋设或回扣伏笔
- **主题**：呈现作品的主题或母题
- **呼应**：首尾呼应、结构对称、意象回收
- **节奏**：情绪转场、呼吸点、节奏调节

示例：
```
E04a | 当晚 | 房间 | 主角 | 拆开信 | E03→ | 困惑 | 推进（第一个主动行动）+ 伏笔（"记得海"的意象首次出现）
```

**叙事目的在游戏改编中的作用：**
- 每个原作事件的叙事目的在游戏中必须被保留——可以换形式（从叙述变成选择、从描写变成 scene），但目的本身不能丢失
- 新增的分支内容也必须有叙事目的——如果一个新节点找不到叙事目的，它不应该存在
- 过渡节点的叙事目的通常是"节奏"类——它们的存在是为了衔接和呼吸

示例（细粒度拆解）：
```
E01   | 开场 | 老公寓走廊 | 主角 | 主角走到家门口，注意到门缝里夹着东西 | → E02 | 日常
E02   | 开场 | 老公寓走廊 | 主角 | 蹲下来，发现是一封没有署名的信 | E01→ E03 | 好奇
E03   | 开场 | 老公寓走廊 | 主角 | 站在走廊里犹豫要不要拆开 | E02→ | 犹豫 ★分支候选
E04a  | 当晚 | 房间 | 主角 | 拆开信，读到"如果你还记得海" | E03→ | 困惑、被触动
```

### 事件间的衔接判定（强制执行）

**每两个相邻事件之间，必须做一次衔接判定。** 不是"发现跳跃才处理"，而是"每一对相邻事件都过一遍"。

判定流程：

```
事件A → [衔接判定] → 事件B

问题1：A和B之间是否存在地点变化？
  → 是 → 标记 [空间过渡]，计划过渡内容
  → 否 → 继续

问题2：A和B之间是否存在时间跳跃？
  → 是 → 标记 [时间过渡]，判断跳跃的时间长度和是否需要补
  → 否 → 继续

问题3：A和B的情绪基调是否有明显变化？
  → 是 → 标记 [情绪过渡]，计划缓冲节点
  → 否 → 继续

问题4：B中是否出现了A中没有交代来源的信息？
  → 是 → 标记 [信息桥接]，计划信息获取节点
  → 否 → 继续

问题5：以上全否，A到B的衔接是否自然？
  → 是 → 标记 [自然衔接]，不需要额外处理
  → 否 → 分析原因，标记处理方案
```

标记格式：
```
E04a | 当晚 | 房间 | 主角 | 拆开信 | ... | 困惑
  [空间过渡] 房间→天台，需补移动过程（楼梯/走廊/开门）
  [时间过渡] 当晚→午夜前，约2-3小时，需交代这段时间做了什么
  [情绪过渡] 困惑→紧张，需一个决心转变的节点
E05  | 午夜前 | 天台 | 主角 | 站在天台上 | ... | 紧张
```

### 章节边界和场景切换的强制检查

在以下位置，衔接判定的优先级更高，必须特别审视：

1. **章节/幕的交界处**：原作换章时往往伴随大的时空跳跃。游戏中即使有 chapterTitle 过渡动画，仍然需要在新章第一个节点的 scene.arrival 和 segments 中做好衔接
2. **major 场景切换处**：每次 scene.id 变化（major 类型），必须确保有过渡——可以写在前一个场景的最后一个节点（"你走出房间"）、新场景的 scene.arrival（"出租车在路上停下来"）、或在中间补一个 transient 过渡节点
3. **角色首次出场处**：新角色出现的节点，前一个节点应该有"即将遇到某人"的暗示或铺垫（听到声音、看到影子、有人提到名字）

### 衔接判定的输出

在事件链的最终输出中，每个衔接判定的结果都要保留。格式：

```
E04a | ... | 困惑
  [空间过渡:需补] 计划2个过渡节点（走廊+楼梯），用transient场景
  [时间过渡:轻描] 在E04a末尾用一句"你坐了很久"带过
  [情绪过渡:需补] 计划1个决心节点（重新看信→做出决定）
E05  | ... | 紧张
```

这些标记会在第三步（结构拆解）中被转化为实际的过渡节点，纳入节拍序列和节点规划。

### 人物层：角色卡

每个有名字的角色（以及重要的无名角色）一张卡。角色卡的详细程度直接决定后续对白写作的质量——写得越具体，后面越不会跑偏。

```
【角色名】

基本信息：
- 身份/职业：
- 年龄段/外貌特征：（只记录影响行为和感知的部分，如"看起来比实际年龄大十岁"）
- 社会位置：（在故事世界中处于什么阶层/圈层/权力关系中）

内在驱动：
- 核心动机：（他最想要什么——一句话）
- 恐惧/软肋：（他最怕什么、什么能刺痛他）
- 自我认知 vs 实际：（他以为自己是什么样的人 vs 他实际的行为模式，如果两者有落差，这个落差就是角色最有戏的地方）
- 决策模式：（遇到压力时习惯怎么做——逃避/硬扛/转移/讨好/攻击/沉默）

说话方式（必须具体到可以直接指导写对白）：
- 句长倾向：（短句为主/中等/偶尔长句/看场合）
- 口语程度：（纯口语/偏口语/书面底色但不端着/正式书面）
- 语气标志：（反问多/陈述多/祈使多/经常用省略句/爱用特定语气词）
- 特殊用词/口头禅：（如果有的话，精确记录）
- 说谎或隐瞒时的表现：（说话方式会有什么变化——变简短、变啰嗦、开始绕弯子、突然客气）
- 情绪激动时的表现：（语序会不会乱、会不会骂人、会不会反而更安静）
- 禁区：（这个角色绝对不会说的话或不会用的表达方式）

行为与弧光：
- 关键行为清单：（在故事中做了哪些重要的事，按时间顺序）
- 弧光：（角色从故事开始到结束发生了什么变化——如果没有变化，标注"无弧光/静态角色"）
- 弧光的外在体现：（这个变化在行为/对白/决策上具体怎么表现出来的——不能写"他成长了"，要写"他在第三章之后开始主动问别人的意见，之前从不这样"）

关系网：
- 与主角的关系：（不只是"朋友/敌人"，而是关系的质感——"表面上互相嘲讽但出事时第一个赶到"）
- 与其他角色的关键关系：
- 关系中的权力动态：（谁更依赖谁、谁更需要谁、谁在关系中占主导）

原文锚点：
- 最能体现说话风格的对白：（2-3句）
- 最能体现性格的行为：（1-2个）
- 最能体现角色张力的场景：（标注场景编号）

首次登场描写素材（供游戏写作时使用）：
- 视觉辨识特征：（2-3个最有辨识度的外貌/穿着/姿态特征。从原文全文中搜集——外貌描写可能散落在不同章节。如果原作完全没有外貌描写，根据角色身份、时代背景、性格合理想象，标注"[想象]"以区分）
- 标志性动作/习惯：（如果有的话——习惯性摸鼻子、总是叼着烟、走路时手插口袋等。这些细节在游戏中反复出现可以强化角色形象）
- 首次出场的场景编号：（标注角色在原作中第一次出现的位置，写作时在对应的游戏节点中插入登场描写）
```

角色卡质量自检：
- [ ] 说话方式是否具体到可以直接指导写对白？不能写"说话很有个性"，要写"喜欢用反问句，很少用完整的句子，经常说到一半就停"
- [ ] 核心动机和恐惧是否清晰到可以预判"这个角色面对某个选择时会怎么做"？
- [ ] 决策模式是否具体到可以在分支中设计"符合角色逻辑的 NPC 反应"？
- [ ] 弧光的外在体现是否具体到可以在 segments 中用视听语言呈现？（不能是"他变了"，必须是可观察的行为变化）
- [ ] 原文锚点是否足够让后续写作时随时校准？
- [ ] 视觉辨识特征是否已收集或想象？是否标注了首次出场位置？（后续写作时需要在首次出场节点插入登场描写）

### 场景层：场景表

每个独立场景一条记录：

```
场景编号 | scene.id | 地点名(scene.name) | 时间 | 在场角色 | 情绪基调 | 感官细节(→scene.description) | 对应事件编号
```

`scene.id` 在索引阶段就要预分配，使用英文下划线格式（如 `old_apartment`、`factory_roof`、`police_station`）。同一个物理地点在不同时间出现时，如果环境感受没有本质变化，保持同一个 id；如果环境感受完全不同了（如白天的办公室 vs 深夜被翻过的办公室），使用不同 id。

感官细节会直接成为游戏中 `scene.description` 的素材，需要写稳定环境特征（光线、声音、空间关系、气味），不写剧情事件。

**首次出现的场景**需要额外准备 `scene.arrival` 素材——玩家第一次来到这个地方时的动作或感受描写（1-2句）。如果原作在该场景没有空间描写，根据时代背景和剧情上下文合理想象，标注"[想象]"。

同样，**关键道具**首次出现时也需要标注外观描写素材：

```
道具编号 | 道具名 | 首次出现的场景编号 | 外观描写（1句） | 后续剧情影响
```

## 不同体量的摄入策略

### 短篇（<1万字）

1. 一次性读入全文
2. 直接生成三层索引
3. 索引总长度控制在原文的 20-30%

### 中篇（1-5万字）

1. 以原作的章节划分为基本单位。如果原作没有明确分章，按场景/时间线自然断点划分为 3-5 个批次
2. 每个章节读入后生成该章的「局部索引」
3. 所有章节读完后，合并局部索引为总索引
4. 合并时检查：跨章节的因果链是否完整、角色弧光是否连贯

### 长篇（>5万字）

1. **以原作章节为基本摄入单位**，不要按字数机械切分——章节是作者划定的叙事单元，强行从中间切断会丢失章节内部的节奏和因果完整性
2. 如果单章过长（超过 15000 字），可以在章节内部按场景转换点再细分，但细分后仍然标记它们属于同一章
3. 如果单章很短（不足 3000 字），可以将相邻的 2-3 个短章合并为一个批次读入，但索引中仍然保留各章的独立标记
4. 每个章节（或章节批次）读入后生成局部索引
5. 每 3-5 章做一次「索引合并与修正」：
   - 合并事件链，补全跨章节的因果关系
   - 更新角色卡（后文的新信息可能修正之前对角色的理解）
   - 合并场景表
   - 校验角色弧光的连贯性（角色在第5章的行为是否与第2章建立的性格一致，如果不一致，是弧光变化还是索引遗漏）
6. 全部读完后做最终合并
7. 索引总长度控制在 4000-6000 字（无论原文多长，但角色卡不计入此限制——角色多的长篇，角色卡可以额外占用空间）

## 索引质量检查

完成索引后，自查：

- [ ] 事件链是否有断裂？（某个事件没有前因或后果）
- [ ] 是否遗漏了伏笔？（后文出现的关键信息，是否在前文的事件中已有铺垫）
- [ ] 角色卡的"说话方式"是否具体到可以指导写对白？
- [ ] 场景表的感官细节是否足以选择 ambient 和 effect？
- [ ] 索引中是否包含了任何心理描写或主观判断？（不应该有——索引记录事实，不记录解读）

## 结构节奏回归校验

索引建完后，必须回归原作验证节奏还原度。索引是压缩，压缩过程中最容易丢失的不是事件本身，而是事件之间的"呼吸"——哪里快、哪里慢、哪里屏息、哪里释放。

### 内部节奏校验（章节/段落级）

回到原作的每个章节，对照索引中对应的事件链段落，检查：

1. **场景密度**：原作本章有几个场景？索引是否完整反映了场景的切换频率？如果原作一章内有 8 个短场景快速交替（高密度叙事），索引不能把它们压缩成 2 个事件——节奏信息会丢失。
2. **停顿与留白**：原作在哪些地方明显放慢了节奏（长段落的环境描写、角色沉默、时间流逝的铺陈）？这些停顿必须在事件链中被标注为"节奏停顿点"，后续分支设计时这些位置不宜插入密集选择。
3. **加速与压缩**：原作在哪些地方明显加速（短句连发、事件密集、对白急促）？这些加速段在后续写作中需要保持高密度的 segments 和紧凑的选项节奏。

在事件链中为每个事件增加一个**节奏标记**：

```
E01 | ... | ○ 慢（铺垫）
E02 | ... | ○ 慢（氛围）
E03 | ... | ● 快（冲突爆发）
E04 | ... | ● 快（追逐/对峙）
E05 | ... | ◐ 停顿（角色独处）
E06 | ... | ● 快（高潮）
E07 | ... | ○ 慢（余韵）
```

标记说明：
- `○ 慢` = 铺垫、氛围、日常、环境描写
- `● 快` = 冲突、追逐、对峙、连续事件
- `◐ 停顿` = 刻意的节奏暂停（留白、沉默、时间跳跃）
- `◑ 转折` = 节奏突变的瞬间（从慢到快或从快到慢）

### 外部节奏校验（全书结构级）

把所有事件的节奏标记串起来，画出全书的"节奏曲线"。用文字描述这条曲线的形状：

```markdown
### 全书节奏曲线

第一幕（E01-E08）：慢启动，逐渐铺垫，E06处第一个小加速
第二幕前半（E09-E15）：节奏均匀推进，E12处有一个停顿（角色回忆）
第二幕后半（E16-E22）：持续加速，事件密集，E20处到达全书最高速度
第三幕（E23-E28）：E23急停，E24-E26缓慢收束，E27-E28余韵
```

然后检查：

- [ ] 原作的整体节奏型是什么？（慢热型 / 开场即高潮型 / 波浪型 / 阶梯上升型）
- [ ] 原作的最高速度段在全书的什么位置？（中段 / 后段 / 多个高潮交替）
- [ ] 原作是否有刻意的"反节奏"设计？（比如高潮之后突然拉回日常，制造反差）
- [ ] 这些节奏特征在分支改编中是否需要保留？（通常答案是"必须保留"）

### 节奏回归的约束力

这份节奏分析不是参考，是约束。后续第三步做分支设计时：

- **慢节奏段**不宜塞入大量分支选择（会打破铺垫的沉浸感）
- **快节奏段**的选项应该简短、紧迫，不宜出现长段叙述
- **停顿点**可以用单选确认来保持参与感，但不应该用多选分支打破安静
- **转折点**是天然的分支候选位置——节奏突变的地方往往也是角色面临选择的地方
- **全书节奏型**决定了 progress 值的分配曲线——慢热型前 30% 的 progress 应该只走到 20 左右

## 输出格式

将索引保存为 markdown 文件，结构为：

```markdown
# 原作索引：《作品名》

## 骨架层：事件链（含节奏标记）
（事件表，每个事件带 ○/●/◐/◑ 节奏标记）

## 节奏分析
### 全书节奏曲线
（文字描述全书的节奏形状）
### 节奏型判定
（慢热型/开场即高潮型/波浪型/阶梯上升型）
### 节奏约束
（哪些位置必须保持慢/快/停顿，不可因分支设计而破坏）

## 人物层：角色卡
（每个角色一个子标题，含首次登场描写素材）

## 场景层：场景表
（场景表，首次出现的场景标注视觉特征摘要）

## 关键道具表
（道具编号 | 道具名 | 首次出现场景 | 外观描写 | 剧情影响）

## 金句与关键句
（从原作中摘录必须保留的原文句子，标注出处事件编号）

## 补充说明
（任何不属于以上各层但对改编重要的信息，如：原作的时代背景、特殊的叙事手法等）
```

### 金句与关键句摘录

在索引构建过程中，标记原作中必须保留的句子。这些句子在游戏改写中应尽可能原样保留，不改写、不缩略、不用自己的话替代。

**需要摘录的句子类型：**
- 角色的标志性台词（读者会记住的那句话）
- 点题句（直接呼应作品标题或核心主题的句子）
- 转折句（改变剧情走向的关键一句话）
- 金句（文学价值高、概括力强、脱离上下文也能独立成立的句子）
- 结尾句（原作最后一句或最后一段）
- 情绪锚点句（全书情感最高浓度的那一两句）

**摘录格式：**
```
| 编号 | 原文 | 出处事件 | 说话者 | 类型 | 在游戏中的预计位置 |
| Q01 | "如果你还记得海" | E02 | 信 | 点题句 | 主线必经节点 |
| Q02 | "我不是来道歉的，我是来还钥匙的。" | E14 | 老邱 | 标志性台词 | 老邱登场节点 |
```

这些句子在后续写作时，应该出现在它们逻辑上应该出现的节点中。如果该句子所在的场景被分支成了多条路径，它应该出现在原作对应的那条路径中（通常是主线/真结局路径）。

## 索引回溯机制（贯穿全流程）

索引不是"第一步做完就放在一边"的东西。它是整个改写流程的中枢文档，后续每一个步骤都必须带入索引并在关键节点回溯校验。

### 什么是回溯校验

在后续步骤的特定时刻，暂停当前工作，重新翻开索引中的对应部分，检查当前产出是否与原作一致。不是泛泛地"看一眼"，是带着具体问题去核对。

### 各步骤的回溯校验点

**第三步（结构拆解）回溯校验：**
- 拆完节拍序列后，对照骨架层事件链，检查：是否有事件在拆解中被遗漏？
- 标记分支点后，对照节奏标记，检查：分支点是否落在了不该打断的节奏位置？（慢节奏段的停顿处不宜插入密集分支）
- 设计汇合点后，对照全书节奏曲线，检查：汇合点是否破坏了原作的加速/减速节奏？

**第四步（状态系统）回溯校验：**
- 设计结局矩阵后，对照事件链的因果关系，检查：每个结局是否都有合理的因果路径支撑？不能凭空出现一个和主线毫无关系的结局。
- 设计 val 变动方案后，对照角色弧光，检查：val 的变化曲线是否与角色弧光的方向一致？

**第五步（分段写作）回溯校验——每写完一章都要做：**
1. 对照角色卡的「说话方式」和「原文锚点」，检查：本章对白是否走形？逐个角色核对。
2. 对照本章对应的事件链段落和节奏标记，检查：
   - 原作中的慢节奏段，在游戏中是否也保持了相对舒缓的 segments 节奏？（不能因为要"增加互动"就把原作刻意放慢的部分塞满选择）
   - 原作中的快节奏段，在游戏中是否也保持了紧凑的推进？（segments 应该更短更密，选项文字也更简短紧迫）
   - 原作中的停顿点，是否用了单选确认而非多选分支？
3. 对照场景表的感官细节，检查：ambient 和 effect 是否与原作场景的感官信息匹配？
4. 对照角色卡的「决策模式」和「自我认知vs实际」，检查：角色在分支中的反应是否符合其行为逻辑？

**第八步（验证）回溯校验：**
- 最终校验时，把全书节奏曲线和 JSON 的 progress 分配对照：progress 的增长曲线是否大致反映了原作的节奏型？（慢热型作品前 30% 的 progress 不应该已经走到 40）

### 回溯的执行方式

每次回溯校验时，在工作记录中用以下格式标注：

```markdown
### 回溯校验：第X章写作完成
- 对照项：角色卡 → ✅ 李明的对白符合短句+反问的规则 / ⚠️ 张华的对白在第3节点偏书面，需修改
- 对照项：节奏标记 → ✅ 慢段保持舒缓 / ⚠️ E12的停顿被跳过
- 对照项：场景感官 → ✅ ambient=rain 与场景表吻合
- 修正措施：（列出需要修改的节点和修改内容）
```

回溯校验发现问题后，**必须修正再继续**，不能标注了问题然后跳过。
